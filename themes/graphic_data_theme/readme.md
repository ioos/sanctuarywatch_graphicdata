This part of the repository contains the WordPress theme used to present the Graphic Data experience on the front end of a WordPress site. While the plugin provides the core content structures, admin tools, and data-handling functionality, the theme controls how that content is displayed to visitors and how the overall experience looks and feels within the site.

## What does the theme do?

The Graphic Data theme provides the visual and layout layer for Graphic Data content in WordPress. It is responsible for page templates, styling, front-end presentation, and the custom site experience that wraps around the plugin’s content types and data-driven features. In practice, the theme helps turn the underlying plugin framework into a polished, user-friendly website experience.

## What is Graphic Data?

Graphic Data is an open-source framework for combining artwork with data in a way that is: 1) easy (and attractive!) for website visitors to use and 2) easy for content managers to maintain. It is intended for small-yet-mighty organizations and teams that have ambitious goals for presenting complex information online, but may not have the resources for large custom web development efforts.

## Is Graphic Data being used anywhere?

Yes. One example is [Sanctuary Watch](https://sanctuarywatch.ioos.us/american-samoa/overview/), a website that presents information used in the management of sanctuaries within the [National Marine Sanctuary System](https://sanctuaries.noaa.gov/).

## How do I learn more about using Graphic Data?

Check out our [guide](https://ioos.github.io/sanctuarywatch_graphicdata/) for more information about setup, usage, and development.

## Relationship to the plugin

This theme is designed to work alongside the Graphic Data plugin. The plugin handles the underlying functionality, content models, and supporting logic, while the theme is responsible for rendering that content on the site and shaping the front-end experience.

## Earlier Work

The general approach taken by Graphic Data is based upon [Infographiq](https://github.com/marinebon/infographiq), created by [Ben Best](https://ecoquants.com/). The broader project also builds on patterns inspired by Joe Sz's [WordPress Plugin Boilerplate Tutorial](https://github.com/JoeSz/WordPress-Plugin-Boilerplate-Tutorial), which is itself based on Devin Vinson's [WordPress Plugin Boilerplate](https://github.com/DevinVinson/WordPress-Plugin-Boilerplate).
